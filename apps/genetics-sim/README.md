# 遺伝のモデル実験 シミュレーター（共有フォルダ版）

中学校3年生・理科「遺伝の規則性と遺伝子」の探Q実習1（教科書 p.10, 23）を、タブレット上で再現するためのWebアプリです。

**ソフトのインストール不要**（Google Chromeは必要）。校内の共有フォルダにファイルを置いて、付属の `.bat` をダブルクリックするだけで動きます。サーバーや Node.js は使いません。

## 仕組み

- 各端末はブラウザの **File System Access API** を使って、共有フォルダ内の `data/` をデータベース代わりに読み書きします。
- 生徒1人につき1つのファイル（`data/students/{id}.json`）を持ち、自分のファイルは自分しか書き換えません。競合は発生しません。
- 申し込み・履歴も全てユニーク名のJSONファイルとして書き出され、各端末が約1.5秒ごとにフォルダをポーリングして同期します。

## ファイル構成

```
GeneticsApp/                   ← 共有フォルダに置くフォルダ
├── index.html                 ← 生徒・先生はこのファイルを開きます
├── style.css
├── app.js
├── README.md
└── data/                      ← 起動時に自動作成されます
    ├── config.json            ← クラスの設定（先生のみ更新）
    ├── students/              ← 生徒の状態（生徒1人＝1ファイル）
    ├── events/                ← 活動の履歴
    └── requests/              ← 交配・交換の依頼（処理後に自動削除）
```

## 動作環境

- **Windows 10 / 11**
- **Google Chrome**（バージョン 86 以降）が各端末にインストールされていること
- 校内のファイル共有（SMB / 共有フォルダ）にアクセスできること
- ※ Microsoft Edge / Firefox / Safari は本ランチャーでは使いません（Chrome 固定）
- ※ 学校で配布されているタブレットの管理者が「共有フォルダへの書き込み」を制限している場合は動きません。事前に1台で動作確認してください。

## 使い方（先生向け）

### 1. ファイル一式を共有フォルダに置く

校内共有フォルダに、以下の **4ファイル** をまとめて置きます。

```
\\school-server\shared\3年理科\GeneticsApp\
├── index.html
├── app.js
├── style.css
└── GeneticsApp起動.bat   ← これが生徒の入口
```

このフォルダは **全生徒が読み書きできる権限** にしておいてください（共有フォルダ自体への書き込み許可は必須ではないが、後述の `data/` サブフォルダ用に書き込み許可が必要）。

> なぜ .bat が必要？
> ブラウザの仕様で `index.html` を共有フォルダから直接ダブルクリックすると、File System Access API が
> 許可ダイアログも出さずに無反応になることがあります。`.bat` は内部で次を行います：
>
> 1. 3ファイルを `%TEMP%\GeneticsApp\` にコピー
> 2. ローカルとして Chrome を起動（**専用プロファイル**で立ち上げるので普段の Chrome に影響しません）
> 3. **Chromeのウィンドウが閉じられたら、コピーしたファイル一式と専用プロファイルを自動削除**
>
> 端末には何も残らず、次回起動時にまた最新版がコピーされます。

### 2. 生徒の操作

1. 共有フォルダ内の **`GeneticsApp起動.bat`** をダブルクリック
   （黒い窓が一瞬出てから Chrome が開きます）
2. 「共有フォルダを選ぶ」をクリック
3. ファイル選択ダイアログで、**手順1の共有フォルダ**（例：`\\school-server\shared\3年理科\GeneticsApp\`）を選ぶ
4. 「このサイトに〜への変更を許可しますか？」→ **「編集を許可」**
5. 名前を入れて参加

### 3. 授業終了時

Chrome のウィンドウを **×** で閉じるだけで OK。バッチが自動的に `%TEMP%` のコピーを削除します。
（Chrome を閉じた後、一瞬黒い窓が残ることがありますが、それが消えれば後片付け完了です。）

### 4. アプリの更新方法

`index.html` `app.js` `style.css` のいずれかを差し替えたいときは、共有フォルダ上の対応ファイルを上書きするだけで OK。
次に生徒が `.bat` を起動したときに自動で最新版がコピーされます。

### 5. 先生として参加

- 名前を入力 → 「先生として参加する」にチェック → パスワード（初期値：`sensei`）を入力して参加。
- パスワードを変える場合は `app.js` の上部にある `TEACHER_PASSWORD = 'sensei'` を編集してください。

### 6. 授業の流れ

| 段階 | 操作 |
| --- | --- |
| 準備 | 先生メニューで「初期遺伝子型」を選んで「全員を初期化」 |
| ステップ1（親→子） | 純系 AA と aa を半分ずつ作りたい場合は、生徒名横の「設定」で個別変更 |
| ステップ2（子→孫） | 「初期遺伝子型 = Aa」で初期化 → 生徒同士でペア交配を繰り返す |
| まとめ | 右パネルでクラス全体の AA : Aa : aa の比、丸 : しわ の比を確認 |
| 後片付け | 「古いデータを掃除」ボタンで `data/` 内のファイルを削除 |

## 使い方（生徒向け）

1. 先生に教えてもらった共有フォルダを開く。
2. **`GeneticsApp起動.bat`** をダブルクリック（Chrome が自動で開きます）。
3. 「共有フォルダを選ぶ」 → 同じフォルダを選択 → 「編集を許可」。
4. 名前を入れて「参加する」。
5. 「ペアで子をつくる」または「遺伝子を交換する」で、相手を選んで操作。
6. 授業が終わったら Chrome のウィンドウを **×** で閉じる（自動で後片付けされます）。

## 教科書の手順との対応

| 教科書（探Q実習1） | アプリの操作 |
| --- | --- |
| 親役の2人がカードを出し合う | 「ペアで子をつくる」モードで相手を選んで申し込む |
| 親役のカードを見ないで選ぶ | お互いの遺伝子から1つずつ **ランダムに** 選ばれる |
| 子の遺伝子の組み合わせを記録 | 結果ダイアログで個別表示／クラス集計に反映 |
| 40回くり返す | 制限なし。ペアを変えて何度でも実施可 |
| 結果の比を表に書く | 右パネルにリアルタイムで表示 |

## トラブルシューティング

### 「共有フォルダを選ぶ」を押してフォルダを選んでも何も起きない／許可ダイアログが出ない
- 共有フォルダの `index.html` を **直接ダブルクリックして開いた** ときに起こる症状です。
- **対処**：いったんブラウザを閉じて、共有フォルダの **`GeneticsApp起動.bat`** をダブルクリックして開き直してください。
- アドレスバーが `file:///C:/Users/.../AppData/Local/Temp/GeneticsApp/index.html` のような形になっていれば OK です。
  `file:////server/...` のような UNC パスのままだと、この症状が再発します。

### ボタンが押せない／ブラウザが対応していないと言われる
- Google Chrome で開いてください（`.bat` 経由で起動すれば自動で Chrome になります）。
- `[エラー] Google Chrome が見つかりません。` と黒い画面が出る場合は、その端末に Chrome がインストールされていません。先生にお願いして Chrome をインストールしてもらってください。

### 「フォルダへのアクセスを拒否しました」
- 共有フォルダへの書き込み権限があるか、先生に確認してください。
- 一度ブラウザを閉じてから開き直して、もう一度「共有フォルダを選ぶ」を試してください。
- ブラウザの許可ダイアログで **「閲覧のみ」を選んでしまった** 可能性があります。「編集を許可」を選び直してください。

### 同期が遅い／反映されない
- ネットワーク（共有フォルダ）の応答が遅い可能性があります。1〜2秒待ってください。
- それでもダメなら、ページを再読込（F5）して、もう一度フォルダを選び直してください。

### 「その名前はすでに使われています」
- 同じ名前の生徒が他にいるか、自分が以前ログインしたまま閉じてしまった可能性があります。
- 1分ほど待つ（30秒で「退出」扱いになる）か、先生にお願いして「古いデータを掃除」してもらってください。

### `data/` フォルダにゴミが溜まる
- 先生メニューの「古いデータを掃除」で全削除できます。授業の終わりに行ってください。

## 制限事項

- ブラウザを閉じてから1〜2分は、その生徒がまだ参加中扱いになります（同名再参加ができないことがあります）。
- ファイル共有のディスク容量・I/O 速度に依存します。1クラス40人くらいまでを想定。
- 生徒どうしが同時に同じ動作を行うと、まれに同期が遅れる場合があります。
