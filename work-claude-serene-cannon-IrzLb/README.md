# 生成物のたな卸し（作品の物置き）

これまで作ってきた **Excel ファイル** や **ブラウザ／iPad で動く HTML アプリ** を、
1 か所にまとめて保存・整理しておくためのリポジトリです。
作品が増えても探しやすいように、種類ごとにフォルダを分けています。

---

## フォルダ構成

```
.
├── README.md        ← このファイル（全作品の目次）
├── apps/            ← ブラウザ／iPad で動く HTML アプリ
│   ├── kanji-note/      かんじノート（漢字まちがい判定つきノート）
│   └── genetics-sim/    遺伝のモデル実験シミュレーター
└── excel/           ← Excel で作った教材・ツール（進度予定表 など）
```

- **1 作品 = 1 フォルダ** が基本です。そのフォルダの中に、アプリ本体も
  配布用の資料（PDF・Word など）もまとめて入れておきます。
- フォルダ名は半角英数（例: `kanji-note`）にしておくと、リンクや共有のときに
  トラブルが起きにくくなります。日本語の正式名称はこの目次で対応づけます。

## 新しい作品を追加するには

むずかしい操作は不要です。次のどちらかでできます。

### A. Claude Code に頼む（おすすめ）

このリポジトリで Claude Code を開き、たとえばこう伝えてください。

> 「席替えアプリの一式を `apps/seat-shuffle/` に入れて、目次にも追加して」
>
> 「この進度予定表の Excel を `excel/` に入れて」

→ フォルダへの保存、この目次への追記、保存（コミット）まで自動でやります。

### B. 自分で置く

1. 種類に合わせて入れる場所を決めます。
   - ブラウザ／iPad で動くもの → `apps/作品名/`
   - Excel ファイル → `excel/`
2. ファイルを置きます。
3. この `README.md` の「作品一覧」に 1 行足します。

## このリポジトリの使い方メモ（GitHub に不慣れな方へ）

- **公開設定**: 教材を人に見せたくない場合は、GitHub 上でこのリポジトリを
  **Private（非公開）** にしておくと安心です。
- **1 つにまとめる利点**: 作品ごとにリポジトリを分けるより、こうして 1 か所に
  フォルダで集めておく方が、後から探したり一覧したりが楽です。
- **HTML アプリの動かし方**: 各アプリのフォルダにある `README.md` に、開き方が
  書いてあります（多くは `index.html` をブラウザで開くだけです）。
